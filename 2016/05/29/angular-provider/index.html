AngularJs依赖注入

怎么说呢，依赖注入是当今框架中的流行元素，依赖注入它是一种软件设计模式，是处理如果让写的程序来得到相应的依赖引用的，通过依赖注入可以有助于代码的维护和测试，对于代码的质量和可重用性得到相当大的提升。

简单的来说，当ng提供服务的过程当中，就会涉及到依赖注入机制，借助于 provider来提供操作，它是被“注入控制器”使用的一个对象，注入机制通过调用一个provider里面的$get()方法，来把得到的内容作为参数在controller里面调用,具体的来拆分，是通过值、工厂、服务、提供者、常值来体现的。

那么下面来细细研究下依赖注入的方式体现：

第一个是factory方法，工厂由$provider提供，module的factory是一个引用，通过调用$get()来进行操作。当一个服务或者控制器需要时，它通常就会派上用场，处理好的值当做参数返回给控制器。


var app = angular.module('myApp', [], function($provide){
  $provide.factory('PP', function(){
    return {'hello': '123'};
  });
});
app.controller('TestController', function($scope, PP){ console.log(PP) });

第二个事service方法，也是由$provider提供，但是这个和factory存在不同之处，他需要提供一个构造方法，来操作，但是factory可以是一个数字或者字符串。 


var app = angular.module('myApp', [], function(){ });
app.service = function(name, constructor){
  app.factory(name, function(){
    return (new constructor());
  });
}

angular 使用这种注入方式，并不要求方法变量与注入名称完全相同，调用的时候会按顺序赋值然后调用就行。

通过上面的例子还会有人困惑，既然factorry和value能够节省那么多的代码，为什么还有人要使用provider，答案是provider允许我们进行一些配置。在前面我们已经提到过当你通过provider（或者其他简写方法）创建一个服务时，你实际上创建了一个新的provider，它将定义你的服务如何被创建。我们没有提到的是，这些provider可以被注入到config函数中，你可以和它们进行一些交互。

首先来说，AngularJS分两个阶段运行你的应用 – config阶段和run阶段。config阶段是你设置任何的provider的阶段。它也是你设置任何的指令，控制器，过滤器以及其它东西的阶段。在run阶段，AngularJS会编译你的DOM并启动你的应用。
